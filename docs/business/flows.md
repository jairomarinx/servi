# User Flows - servi.cam

## Flow 1: Provider onboards and creates a service
1. Register (email + password)
2. Verify email
3. Complete person profile
4. Activate provider role
5. Add skills
6. Create service
7. Service published and searchable

## Flow 2: Client books a service directly
1. Register or login
2. Search by category and city
3. Compare providers (rating, price, trust score)
4. Book service
5. Provider confirms
6. Service delivered
7. Client reviews provider

## Flow 3: Client posts to the Feed
1. Client creates service request
2. System checks for matching services
3. No match found: published to Feed
4. Providers see request ranked by skill match
5. Provider submits proposal
6. Client accepts proposal
7. Booking created
8. Service delivered
9. Both parties review each other
